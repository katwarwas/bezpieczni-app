SQL Injection to rodzaj cyberataku, który polega na wstrzyknięciu złośliwego kodu SQL do zapytania kierowanego do bazy danych. Celem ataku jest manipulacja zapytaniami SQL w taki sposób, aby uzyskać nieautoryzowany dostęp do danych, ich modyfikację lub usunięcie.

Jak działa SQL Injection?

Manipulacja zapytaniami SQL

Złośliwy kod zmienia pierwotne zapytanie SQL w taki sposób, że baza danych wykonuje nieautoryzowane polecenia.

Przykład:
Zapytanie SQL: "SELECT * FROM users WHERE username = 'admin' AND password = 'password';"
może zostać zmienione na: "SELECT * FROM users WHERE username = 'admin' OR '1'='1' AND password = 'password';
co zawsze zwróci prawdziwy wynik, pozwalając na zalogowanie bez poprawnego hasła.

Wstrzyknięcie kodu

Atakujący wprowadza złośliwy kod SQL w polach wejściowych aplikacji, takich jak formularze logowania, pola wyszukiwania, URL-e lub nagłówki HTTP.

Przykład:
W polu login użytkownik zamiast standardowego „admin” wpisuje: admin' OR '1'='1

Uzyskanie dostępu do danych

Atakujący może wykorzystać SQL Injection do przeglądania tabel, wydobywania danych, wstawiania nowych rekordów, modyfikowania istniejących danych lub usuwania danych.

Typy SQL Injection

Classic SQL Injection

Wstrzyknięcie kodu w polach wejściowych, które następnie są bezpośrednio używane w zapytaniach SQL.

Blind SQL Injection

Stosowany, gdy aplikacja nie zwraca wyników zapytań SQL, ale odpowiedzi serwera (np. czas odpowiedzi lub różne kody błędów) mogą pomóc w dedukcji struktury bazy danych.

Error-based SQL Injection

Atakujący wstrzykuje kod SQL, który celowo powoduje błędy w bazie danych, aby uzyskać informacje o jej strukturze.

Union-based SQL Injection

Wykorzystanie operatora SQL UNION do łączenia wyników zapytań, co pozwala na wydobycie dodatkowych danych.
